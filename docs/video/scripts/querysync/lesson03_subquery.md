### [02:30] Step 1: サブクエリのイメージ — SELECTの中にSELECT

**【画像 No.3】**（外側のSELECTと内側のSELECTの図解）**【画像 No.4】**（employeesテーブル見本）

**めたん**

サブクエリは、**かっこ《()》でくくった SELECT 文**を、ほかの SELECT の「値が入る場所」に書くの。例えば《WHERE salary > (SELECT AVG(salary) FROM employees)》みたいに。このとき、**内側の《(SELECT AVG(salary) FROM employees)》が先に実行されて、平均給与がひとつの値になる**。その値と《salary》を比較して、「平均より大きい行だけ」を外側の SELECT が取り出す、って流れね。

**ずんだもん**

内側が先に実行されて、その答えを外側が使うんだ。

**めたん**

そう。だから「**内側の SELECT が返す結果**」が、外側から見ると「ひとつの値」だったり「複数行」だったりする。返す結果の形によって、WHERE で《=》や《>》で使うか、《IN》で使うか、あるいは FROM のうしろに「表」として書くか、が決まってくるよ。まずは「かっこでくくった SELECT が、外側のクエリのどこかで使われる」って形を頭に入れておこう。

**ずんだもん**

かっこでくくった SELECT が、外側で使われる。分かった。

**めたん**

では、「サブクエリ」がどこに書かれているか、次の文のなかで探してみて。《SELECT * FROM employees WHERE salary > (SELECT AVG(salary) FROM employees)》

**めたん**

**【演習キュー】ここで手を動かしてみましょう！**
一度止めて、自分で考えてみて。この文のなかで、**サブクエリはどこか**。かっこでくくられた SELECT がどれか、指さしてみて。答えはこのあとすぐ見せるから。

### [05:30] Step 1 の答え・解説 ＋ Step 2 へのつなぎ

**【画像 No.3】**（内側のSELECTがWHEREの値になる図）

**めたん**

正解は、《(SELECT AVG(salary) FROM employees)》の部分だよ。このかっこのなかの SELECT がサブクエリ。ここが先に実行されて「平均給与」がひとつの値になり、その値と《salary》を比較しているね。

**ずんだもん**

WHERE のうしろで、比べる「もう片方」がサブクエリになってるんだ。

**めたん**

そう。こういう「**ひとつの値（1行1列）**」を返すサブクエリのことを、**スカラーサブクエリ**って呼ぶことがあるよ。《>》や《=》で比較するときは、内側は**必ず1行1列**になるように書く。次は、この「平均より高い給与の人」を実際に書く練習をしよう。

---

### [08:00] Step 2: WHERE句で使う — スカラーサブクエリ（比較）

**【画像 No.4】**（employeesテーブル見本）**【画像 No.5】**（WHERE と スカラーサブクエリの構文図解）

**めたん**

《WHERE 列名 比較演算子 (SELECT …)》の形で、「**その列の値が、サブクエリの結果と比較して真になる行だけ**」を取り出すの。《SELECT AVG(salary) FROM employees》は「全社員の平均給与」を1行1列で返すから、《WHERE salary > (SELECT AVG(salary) FROM employees)》と書けば、「平均給与より高い社員」だけが取れるよ。

**ずんだもん**

平均を自分で計算しなくていいの、便利だね。

**めたん**

うん。条件に「いまのデータから出した値」を使いたいときに、サブクエリで書いておけば、データが変わっても自動でやり直してくれるの。では、**平均給与より高い給与をもらっている社員**を、全カラムで取得する文を考えてみて。

**めたん**

**【演習キュー】ここで手を動かしてみましょう！**
一度止めて、自分で書いてみて。平均給与より高い給与の社員を、全カラムで取得する文を考えてみて。答えはこのあとすぐ見せるから。

---

### [11:00] Step 2 の答え・Step 3 へのつなぎ

**【画像 No.4】**（結果のイメージ・該当する行だけ）

**めたん**

正解の例は《SELECT * FROM employees WHERE salary > (SELECT AVG(salary) FROM employees)》だよ。内側で平均が求まって、その値より大きい《salary》の行だけ出てくるはず。

**ずんだもん**

出た！ サブクエリで条件が書けた。

**めたん**

いいね。次は、**「ひとつの値」じゃなくて「複数行」**を条件に使うパターン。例えば「**開発部と営業部に所属してる社員だけ**」出したいとき、部署IDの一覧をサブクエリで出して、《WHERE department_id IN (その結果)》みたいに書くの。《IN》は「この列の値が、かっこのなかのどれかと一致した行」を残すよ。

---

### [13:00] Step 3: IN とサブクエリ — 複数行を条件に

**【画像 No.6】**（departments と employees の関係）**【画像 No.7】**（IN (SELECT ...) のイメージ図）

**めたん**

《WHERE 列名 IN (SELECT 列名 FROM テーブル …)》と書くと、**サブクエリが返す複数行の「その列の値」のどれか**に、外側の列の値が一致した行だけ残るの。例えば「部署名が『開発部』か『営業部』の id」を《SELECT id FROM departments WHERE name IN ('開発部', '営業部')》で取ってきて、《WHERE employees.department_id IN (SELECT id FROM departments WHERE name IN ('開発部', '営業部'))》とすれば、開発部と営業部の社員だけ取れるよ。

**ずんだもん**

IN のなかに SELECT を入れると、その結果のどれかと一致、ってことだね。

**めたん**

そう。サブクエリが**複数行1列**を返すときは、《=》じゃなくて《IN》を使う。では、**部署名が『開発部』の社員**を、全カラムで取得する文を考えてみて。departments から「開発部」の id を取るサブクエリと《IN》を使ってね。

**めたん**

**【演習キュー】ここで手を動かしてみましょう！**
一度止めて、自分で書いてみて。部署名が『開発部』の社員を、employees から全カラムで取得する文を考えてみて。答えはこのあとすぐ見せるから。

---

### [16:00] Step 3 の答え・Step 4 へのつなぎ

**めたん**

正解の例は《SELECT * FROM employees WHERE department_id IN (SELECT id FROM departments WHERE name = '開発部')》だよ。サブクエリで「開発部」の id を取って、その id に一致する《department_id》の社員だけ残してるね。

**ずんだもん**

IN でサブクエリ、覚えた。

**めたん**

うん。最後に、**FROM のうしろ**にサブクエリを書くパターン。SELECT の結果を**ひとつの表のように見なして**、さらにその上で SELECT したり JOIN したりするやり方だよ。

---

### [17:30] Step 4: FROM句のサブクエリ — 派生テーブル

**【画像 No.8】**（FROM (SELECT ...) のイメージ・結果が表になる）

**めたん**

《FROM (SELECT …) AS 別名》と書くと、**かっこのなかの SELECT の結果が、ひとつのテーブルとして扱われる**の。これを**派生テーブル**って言うことがあるよ。別名《AS》は必ず付けることになってる。例えば「部署ごとの平均給与」を出した結果を、さらに「その平均給与が 50 万以上の部署だけ」に絞りたいとき、《FROM (SELECT department_id, AVG(salary) AS avg_sal FROM employees GROUP BY department_id) AS dept_avg》みたいに書いて、そのうしろで《WHERE avg_sal >= 500000》とかできるの。

**ずんだもん**

いったん集計した結果を、表だと思ってもう一回クエリするんだね。

**めたん**

そう。「まずサブクエリで集計や絞り込みをした結果」を表に見立てて、外側でもう一度 SELECT する、っていう形。では、《employees》から「部署ごとの平均給与」を出すサブクエリを FROM に書いて、その結果に《dept_avg》って別名を付けるところまで、自分で書いてみて。

**めたん**

**【演習キュー】ここで手を動かしてみましょう！**
一度止めて、自分で書いてみて。FROM のうしろに「部署ごとの平均給与」を出すサブクエリを書き、《AS dept_avg》で別名を付けた文を考えてみて。外側の SELECT は《SELECT * FROM (…) AS dept_avg》でいいよ。答えはこのあとすぐ見せるから。

---

### [20:00] Step 4 の答え・まとめ

**めたん**

正解の例は《SELECT * FROM (SELECT department_id, AVG(salary) AS avg_sal FROM employees GROUP BY department_id) AS dept_avg》だよ。FROM のうしろのかっこがサブクエリで、その結果が《dept_avg》という名前の表として扱われるね。

**ずんだもん**

FROM にもサブクエリ書けるんだ。長いけど仕組みは分かった。

**【画像 No.9】**（まとめ：サブクエリの種類と書き場所）

**めたん**

いいね。今日のまとめだよ。
**サブクエリ**は「SELECT のなかに、かっこでくくったもうひとつの SELECT を書く」こと。**WHERE で使う**ときは、**1行1列**なら《>》や《=》で比較、**複数行**なら《IN (SELECT …)》で「どれかと一致」を書く。**FROM で使う**ときは《FROM (SELECT …) AS 別名》で、結果を表のようにして外側の SELECT でさらに扱える。**内側が先に実行されて、その結果を外側が使う**って流れを忘れないでね。

**ずんだもん**

内側が先、外側がそれを使う。覚えた。

**めたん**

次は《集計やウィンドウ関数》とか、もっと便利な集計のやり方をやっていくから、楽しみにしててね。また次の動画で会おう。

**ずんだもん**

またねー！

## 編集メモ（VOICEVOX / CapCut 用）

- **めたん**: 解説は落ち着いたトーン、まとめは少し明るく。優しく教える感じ。
- **ずんだもん**: 生徒役。「なるほど」「おけ」など反応は少し大きめ。不安→理解→喜びの流れを意識。
- **テロップ強調**: サブクエリ, WHERE, IN, FROM, スカラー, 派生テーブル, AS, AVG, employees, departments。重要な用語は《》で囲んだ部分をそのままテロップに。
- **演習キュー**: 毎回同じデザインで統一。視聴者が一時停止して手を書くタイミングなので視認性を高く。
- **画像差し込み**: 台本中の【画像 No.X】は、そのセリフ・ブロックで表示する図解の指示。No.10（めたん）・No.11（ずんだもん）はセリフに応じて小窓・カットインで全編に差し込む。
- **BGM**: 小さめ。技術解説なので主張しすぎないこと。
- **尺**: 解説＋掛け合いでゆったり。無理に短くせず、腹落ちするペースで。
