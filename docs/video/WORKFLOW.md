# おもしろ動画制作 作業手順書

**めたん**（講師役）と **ずんだもん**（生徒役）を登場人物に、**VOICEVOX（VOICEBOX）** と **CapCut** を使って、Movidrill 用の「観ていて楽しい・身につく」技術解説動画を作る手順をまとめます。

---

## 1. 方針：なぜ「おもしろく」するか

- **飽きずに最後まで観てもらう** → 離脱を減らし、演習ステップまで到達してもらう。
- **キャラクターに親しみを持ってもらう** → めたんが教え、ずんだもんが「分かった／難しい」と反応するやり取りで「一緒に学んでいる」感を出す。
- **技術内容は正確に、でも堅くしすぎない** → 専門用語はそのまま、説明の言い回しや間で緩急をつける。

---

## 2. 使用ツール

| ツール | 用途 | 備考 |
|:--|:--|:--|
| **VOICEVOX** | めたん・ずんだもんの音声合成 | 無料。VOICEBOX 表記の場合は本手順では VOICEVOX を想定 |
| **CapCut** | 動画編集（字幕・BGM・切り替え・テロップ） | 無料版で十分対応可能 |
| **台本（Markdown）** | 本番用テキスト・タイムスタンプ・メモ | `docs/video/scripts/` で管理 |

---

## 3. 作業の流れ（全体）

```
1. 企画・台本作成  →  2. 音声生成（VOICEVOX）  →  3. 編集（CapCut）  →  4. 書き出し・アップロード
```

各ステップで「おもしろさ」を入れるポイントを以下にまとめます。

---

## 4. ステップ1：企画・台本作成

### 4.1 台本の置き場所

- **新規レッスン** は `docs/video/scripts/` 配下に作成する。
- テンプレートは `scripts/_template.md` をコピーして使う。
- モジュール別にフォルダを分けると管理しやすい（例: `scripts/querysync/lesson01-select-basic.md`）。

### 4.2 おもしろくするための台本のコツ

1. **役割を分ける（登場人物）**
   - **めたん**: **講師役**。解説・説明・「ここで手を動かしてみましょう」のキューを担当。落ち着いたトーンで教える。
   - **ずんだもん**: **生徒役**。視聴者と同じ目線で「なるほど」「ここ難しいね」「やってみるね」と反応したり質問したりする。一緒に学んでいる感を出す。

2. **言い回し**
   - 一文を短くする。長い説明は2〜3文に分割。
   - 「〜なんだ」「〜してみよう」「ここ、ちょっと手を動かそう」など、話し言葉を入れる。
   - 重要な用語の前後に少し間を入れる（台本に「（一拍）」などとメモしておく）。

3. **演習タイミングをはっきり書く**
   - 「ここで手を動かしてみましょう」の** exact なセリフ**を台本に書く。
   - Movidrill のステップ切り替えと動画のタイムスタンプが対応するので、このキューは変えない。

4. **タイムスタンプを台本に書く**
   - 例: `[00:00]` イントロ、`[01:30]` Step1 開始、`[03:45]` 「ここで手を動かしてみましょう」。
   - 後で CapCut で字幕やチャプターを合わせるときの基準になる。

5. **ネタ・共感ポイント（任意）**
   - ずんだもん（生徒役）に「SELECT って最初なんか怖くない？」「大丈夫、めたんが教えてくれたからやってみる」のようなセリフを入れると親しみが出る。
   - やりすぎない程度に、モジュールの雰囲気に合わせて入れる。

### 4.3 チェックリスト（台本完了時）

- [ ] 1動画あたり 3〜7 ステップになっているか
- [ ] 各ステップの「ここで手を動かしてみましょう」が入っているか
- [ ] タイムスタンプ（目安でOK）が書いてあるか
- [ ] 解答の説明は「次のチャプター冒頭」に回す構成か（PROJECT_PLAN のルール）

## 5. ステップ2：音声生成（VOICEVOX）

### 5.1 準備

- [VOICEVOX](https://voicevox.hiroshiba.jp/) をインストールし、**めたん**と**ずんだもん**の音声をダウンロードする。
- 台本を「話者ごと」「ブロックごと」に分割する（長いと調整しづらいため）。

### 5.2 おもしろくするための音声のコツ

1. **話速・抑揚**
   - 重要な用語の前は少しゆっくり、慣れてきたら少し速くなど、メリハリをつける。
   - VOICEVOX の「話速」「音高」を少し変えるだけで印象が変わる。

2. **間（ポーズ）**
   - 台本に「（一拍）」と書いたところは、VOICEVOX では別クリップとして「無音」や短いポーズを CapCut 側で入れる想定でもよい。
   - または、VOICEVOX で改行を入れて別セリフにし、その間を長めに取る。

3. **感情・ノート**
   - **めたん**: 解説中は落ち着きめ、まとめでは少し明るめなど使い分ける。
   - **ずんだもん**: 生徒役なので「分かった」「不安」「やってみる」など感情をメモ（`[ノート: 嬉しい]` 等）しておくと編集時に楽。

### 5.3 書き出し

- セリフごとに **WAV または MP3** で書き出す。
- ファイル名は話者と順序が分かるようにする（例: `metan_001_intro.wav`、`zundamon_002_reaction.wav`）。
- 後で CapCut でタイムラインに並べるので、番号を振っておくと効く。

---

## 6. ステップ3：編集（CapCut）

### 6.1 基本の並べ方

1. **音声トラック**
   - VOICEVOX で出力したクリップを時系列で並べる。
   - 必要なら無音や環境音で「間」を調整。

2. **映像トラック**
   - 画面収録（コード・ターミナル・図解）やスライド、イラストを配置する。
   - めたん（講師）・ずんだもん（生徒）の立ち絵やアイコンを話者に合わせて小窓で出すと「授業を受けている」感じが出る。

3. **テロップ・字幕**
   - 重要な用語（SELECT、WHERE、API など）はテロップで補強する。
   - 「ここで手を動かしてみましょう」は**毎回同じデザイン**にすると、Movidrill と連動したときの視聴者の認識が揃う。

### 6.2 おもしろくするための編集のコツ

1. **カットとリズム**
   - 長い説明が続くときは、数秒ごとに画面を切り替える（コード→図解→めたん／ずんだもんなど）と飽きにくい。

2. **BGM**
   - 小さめの音量で、明るくても邪魔にならない曲を選ぶ。技術解説なので主張しすぎないこと。

3. **チャプター（タイムスタンプ）**
   - 動画の説明欄に書くチャプターと、台本のタイムスタンプを一致させる。
   - Movidrill の `videoTimestamp`（秒）は、このチャプターを元にレッスンJSONに記録する。

4. **サムネイル**
   - めたん＋ずんだもん＋技術テーマ（SQL、Docker など）が分かる構図にするとクリックされやすい。

### 6.3 チェックリスト（編集完了時）

- [ ] 各ステップの「ここで手を動かしてみましょう」の前後でチャプターが切れているか
- [ ] 解答の説明は次のチャプター冒頭になっているか
- [ ] 音声と字幕・テロップのずれがないか
- [ ] チャプター一覧（タイムスタンプ）をメモし、レッスンJSON用に秒数で控えたか

---

## 7. ステップ4：書き出し・アップロード

1. **CapCut で書き出し**
   - 解像度・フレームレートはチャンネル方針に合わせる（例: 1080p 30fps）。
   - ファイル名にモジュール名・レッスン名を入れておくと管理しやすい。

2. **YouTube にアップロード**
   - タイトル・説明文にキーワードを入れる。
   - **説明欄にチャプター（タイムスタンプ）** を記載する。

3. **Movidrill 用データの更新**
   - レッスンJSON の `videoId` を新しい動画IDに更新する。
   - 各ステップの `videoTimestamp` を、動画の実際の秒数に合わせて更新する。

## 8. モジュール別の「おもしろさ」の例（アイデア）

| モジュール | おもしろくするポイント例（めたん＝講師・ずんだもん＝生徒） |
|:--|:--|
| QuerySync | めたんが SELECT を説明 → ずんだもん「最初は怖いけど、一緒に書いてみるね」「WHERE で一気に広がった！」 |
| APISync | めたん「API はお店の窓口みたいなもの」→ ずんだもん「リクエストとレスポンス、やってみる」 |
| PipeSync | めたん「CI/CD は自動でお手伝いしてくれる人」→ ずんだもん「便利そう、設定してみたい」 |
| InfraPlay | めたんがサーバー構築を解説 → ずんだもん「1台から増やす感覚、やってみよう」 |
| ContainerSync | めたん「Docker は荷物をまとめた箱」→ ずんだもん「その箱を動かすイメージだね」 |
| DesignSync | めたんが設計図の描き方を解説 → ずんだもん「自分で描くと頭に入る」 |
| NetMap | めたんがパケットの流れを説明 → ずんだもん「図で追いかけてみる」 |
| HackLab | めたんが攻撃・防御を解説 → ずんだもん「攻撃の気持ちになると防御の大切さが分かる」 |

---

## 9. まとめ

- **台本**で「めたん（講師）・ずんだもん（生徒）の役割」と「演習キュー・タイムスタンプ」を決める。
- **VOICEVOX**でめたん・ずんだもんの声にメリハリと間をつける。
- **CapCut**で映像・テロップ・チャプターを整え、Movidrill のステップと対応させる。

この流れを繰り返し、各モジュールの台本を `scripts/` 配下に増やしていくことで、**めたんに教わりながらずんだもんと一緒に学べる、おもしろくて身につく動画**を効率的に作れます。
